Dynalinks претерпел важные изменения.

Во-первых, из него извлечено всё, связанное с сохраннением в файлы. Теперь это отдельный объект Dynalinks_File_Proxy. Туда же функцию SaveAs, она всё равно работает только внутри вкладок. А попадая в неподобающий контекст, вызывает кучу синтаксических ошибок.

create_dynalinks
Это что-то вроде прокси объекта для связи базы данных с browser.storage.

Так значит у нас два прокси - один для файлов, другой для storage, будут ещё другие.

proxy_data
Жуткая шняга, связанная с port-connect-message схемой, то есть обменом сообщениями через порты. Вся эта штука плодит кучу шаблонного кода и плохо продумана. Не мной, разрабами API. 

dynalinks.min.js
Собственно, движок данных. В нём мало что осталось, только разбор (parsing) базы данных. Но его ещё надо переделать. Надо разобраться с избранным. Все эти записи надо выбросить в отдельный объект, где они и будут существовать. Ключи favorite и favorite_text у простых записей надо уничтожить. Связь между записью в избранном и обычной базе будет проходить через id.


link_view_run.js
Наконец, выбросил сюда запуск графического движка таблицы ссылок. Это правильно.
